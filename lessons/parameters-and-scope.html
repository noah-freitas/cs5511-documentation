JavaScript Functions — Parameters and Scope

Getting data to your functions.

Introduction

Scope in a programming language controls the visibility and lifetimes of variables and parameters. Douglas Crockford, JavaScript: The Good Parts

Without scope in a programming language, building an application of any complexity would be impossible. This is because scope allows a program to be broken up into different parts that are hidden from one another. If this sounds very similiar to our discussion of functions, it is because scope and functions are very closely connected — especially in JavaScript!

Scope in JavaScript

JavaScript basically has one mechanism for creating a new scope: functions. Each function has its own scope, which contains its parameters and local variables. In addition to its own scope, a function also has access to its containing scopes, all the way up to the global scope.

The containing scope of a function is determined by the context in which the function is defined not where it is called. In computer science terminology, this means that JavaScript is lexically scoped. While many of the implications of this are beyond the scope of this class, it is important to have some familiarity with these concepts as they form the basis for many advanced JavaScript techniques including information hiding with closures.

Terminology

Scope
A set of rules that determines where a variable is accessible from a particular part of your program.
Parameter
A named value passed to a function.
arguments
A special array-like object available whenever a function is called in JavaScript. It contains the parameters actually passed to the function when it was invoked.

Usage and Syntax

At a Glance

var myFunction = function(parameter) {
    var returnValue;
    
    // Compute return value.
    
    return returnValue;
};

Syntax in Depth

JavaScript supports two syntaxes for creating functions:

Function Expressions
var myFunction = function() {};

Function expressions provide much greater flexibility and control over the architecture of your code.

Pros:
  • Functions can be assigned anywhere a value can be (variable, object property, etc.).
  • The syntax highlights the fact that functions are just objects in JavaScript.
Cons:
  • Functions are not hoisted. They must be implemented before they are used.
Function Declarations
function myFunction() {}

This syntax follows the form of function declaration in many other C-based languages.

Pros:
  • Functions are hoisted (see above), so they can be called before they are implemented.
Cons:
  • Most functions created with this syntax are added to the global namespace, which is considered a poor design practice. (Inner functions are an exception to this rule.)
  • For C language programmers, this syntax obscures the dynamic nature of JavaScript functions.

Take Aways

  • Use functions whenever you are going to repeat code.
  • Create your functions using the Function Expression syntax.
