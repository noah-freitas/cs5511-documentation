JavaScript Functions — Creating Functions

Learn how and why to create functions in your JavaScript code.

Introduction

Generally speaking, a function is a "subprogram" that can be called by code external (or internal in the case of recursion) to the function. Like the program itself, a function is composed of a sequence of statements called the function body. Values can be passed to a function, and the function can return a value. Mozilla Developer Network

Functions allow us to break up difficult problems into more manageable, maintainable, and testable pieces. Generally a JavaScript application will be composed of a number of small functions which are called either when the application loads or in response to user initiated events.

Most functions are designed as "black-boxes" which perform a single task and hide the way that they perform this task from the rest of the application.

When Would I Use a Function?

Suppose you are building a weather application that needs to convert from fahrenheit to celsius degrees during different stages of its execution. If you rewrite the conversion formula everytime you need to convert units, your code will be harder to maintain and test.

It will be harder to maintain because it is implemented multiple times in your application, which means that if you want to change the number of significant digits that are displayed for the converted value, you will have to find every time you convert in your application and change it. It will be harder to test because you cannot pull out a single set of statements and unit test them in isolation from the rest of the application.

By creating a function, you solve both these problems and make your code easier to read.

var toCelsius = function(fahrenheit) {
    var celsius = (fahrenheit - 32) * 5/9;
    return Math.round(celsius);
};

Terminology

Usage and Syntax

At a Glance

Syntax in Depth

JavaScript supports two syntaxes for creating functions:

Function Expressions
var myFunction = function() {};

Function expressions provide much greater flexibility and control over the architecture of your code.

Pros:
  • Functions can be assigned anywhere a value can be (variable, object property, etc.).
  • The syntax highlights the fact that functions are just objects in JavaScript.
Cons:
  • Functions are not hoisted. They must be implemented before they are used.
Function Declarations
function myFunction() {}

This syntax follows the form of function declaration in many other C-based languages.

Pros:
  • Functions are hoisted (see above), so they can be called before they are implemented.
Cons:
  • Most functions created with this syntax are added to the global namespace, which is considered a poor design practice. (Inner functions are an exception to this rule.)
  • For C language programmers, this syntax obscures the dynamic nature of JavaScript functions.

Take Aways

  • Use functions whenever you are going to repeat code.
  • Create your functions using the Function Expression syntax.
