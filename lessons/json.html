JSON

Using JavaScript to transmit your data.

Introduction

JSON (JavaScript Object Notation) is a lightweight data-interchange format. json.org

Due to its simplicity, human-readability, and support in many programming languages, JSON has has become the de facto standard for transmitting data on the web. In so doing, it has supplanted competing technologies, such as XML.

Very Brief History

Douglas Crockford first "discovered" JSON in 2001, and formally specified it in a RFC in 2006. YAHOO! (Crockford's employer at the time) begun using JSON for some of its web service APIs begining in 2005, and Google followed in 2006.

JSON vs. JavaScript

JSON is a subset of JavaScript (specifically ECMAScript 3). So the immediate question for those who know JavaScript is "How much has been removed from the subset?"

The answer to that question encompasses two categories of the language: its value types and syntax.

Value Types
A few JavaScript value types and built in values are not present in JSON:
  • undefined
  • Function
  • Number Formats and Values:
    • NaN (null in JSON)
    • Infinity (null in JSON)
    • Hexidecimal Format
    • Octal Format
Syntax
JSON has a more limited syntax than JavaScript:
Strings
All string values must be enclosed in double quotes. Single quotes are not allowed.
Objects
Only the object literal syntax is supported. This should make sense since JSON does not have functions.
Object keys must be enclosed in double quotes.
Arrays
Just like Objects, only the Array Literal syntax is supported.

The window.JSON Object

All modern browsers (IE 8+) have a built-in object for creating and working with JSON in JavaScript. This built-in object replaces older methods of parsing and creating JSON, including third party libraries and using the eval function.

The JSON object has two methods: stringify and parse. As should be obvious from their names, stringify takes a JavaScript object and turns it into a JSON string, and parse takes a JSON string and returns a JavaScript object.

JSON Example

var person = {
    name: 'John Doe',
    age: 43,
    profession: {
        industry: 'IT',
        yearsOfExperience: 17,
        position: 'Manager'
    },
    sayHello: function () {
        return 'Hello, my name is John Doe';
    },
    children: ['John Jr.', 'Mary']
};

var personJson = JSON.stringify(person);

// personJson now contains a JSON string with the following stucture:
/*
{
    "name": "John Doe",
    "age": 43,
    "profession": {
        "industry": "IT",
        "yearsOfExperience": 17,
        "position": "Manager"
    },
    "children": [
        "John Jr.",
        "Mary"
    ]
}
*/

var clone = JSON.parse(personJson);
// clone now has copies of all the same properties that person does, except
// for the sayHello method.

Notice a few things about the result of stringifying person:

  • The function sayHello was ignored.
  • All the object keys have been quoted.
  • The string values, which we declared using single quotes are now wrapped in double quotes.
  • The number values have been preserved as numbers.

Take Aways

  • JSON is a subset of JavaScript used for data interchange across the web.
  • The most significant difference between JSON and JavaScript is that JSON does not have functions.
  • Modern web browsers have a built-in object for moving data between JSON strings and JavaScript objects.
