User-driven programming

To add actual useful functionality to a web page, we need to do more than inspecting and modifying the document — we must be able to detect the user"s actions and respond to them. This is done by handling events. Marijn Haverbeke, Eloquent Javascript: A Modern Introduction to Programming</a>

What is a Javascript event?

In programming, an event is like 'potential energy' — like a spring — a set of statements that will execute only under a specific condition. Unlike an IF statement, which is a conditional branch within the code, an Event can be invoked automatically by conditions outside of the code — such as a user interacting with a browser or page element.

Events are an important part of what makes Javascript so popular, they enable user-interactive programming. Events are simple to implement and we have already seen some examples of events in our projects. For traditional computer programmers using procedure or object-oriented approaches, event-driven programming may be a slightly new concept.

Event Listeners

In general, I find it useful to think about events as both event listeners and event handlers. The 'listener' is the code that 'sets' the event and the 'handler' is the code that runs when the event 'fires.'

In Javascript, it is very simple to instantiate event listeners by assignment to the DOM. In the past, different browsers provided a different interface to their event model, but now all current browsers work with the W3C DOM specification. In the most basic case, you can pass a short Javascript statement into the HTML attribute. 

<a href="#" onclick="alert('Clicked')">

You can also have Javascript assign the event handler to the DOM element property. In this case we are using a Javascript anonymous function.

var listener = document.getElementById("listener");
listener.onclick = function() {..};

In this case we create a named function and assign the function name to the event listener.

var listener = document.getElementById("listener");
listener.onclick = clickMe;
function clickMe() {...};

Event Handlers

An event handler is essentially the code that runs in the event of the event. Typically this would be implemented as a Javascript function, i.e. in the functions we defined above.

The DOM event model provides access to the event"s TARGET. The target is the element which triggered the event. 

var t = document.getElementById("t");
var pa = document.getElementById("pa");
t.onclick = h;
function h(event) {
    pa.innerHTML = event.target.innerHTML;
}
